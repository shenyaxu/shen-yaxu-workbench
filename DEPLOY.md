# 心理工作台 -- 网上部署教程

> 这份教程会手把手教你，把工作台从自己的电脑"搬"到互联网上，这样你用手机、用其他电脑也能随时访问。

不要被技术名词吓到，跟着步骤一步步做就行！

---

## 前置准备：你需要两个账号

在正式开始之前，你需要注册两个网站账号。它们都是免费的。

### 1. GitHub 账号（用来存放代码）

GitHub 就像是一个"网盘"，但专门用来存放程序代码。

**注册步骤：**

1. 在浏览器打开 https://github.com/signup
2. 填写你的邮箱地址，点击 "Continue"
3. 设置一个密码，点击 "Continue"
4. 填写一个用户名（英文的，比如 `shenyaxu`），点击 "Continue"
5. 按照提示完成验证（勾选确认、验证邮箱等）
6. 注册完成后，记住你的**用户名**和**密码**

> 如果已经有账号了，直接用就行，跳过这步。

---

### 2. Render 账号（用来把代码"运行"起来，变成网站）

Render 是一个免费的服务，它可以把你的代码变成一个可以访问的网站。我们后面会详细用它。

**注册步骤：**

1. 在浏览器打开 https://dashboard.render.com/register
2. 推荐点击 **"Sign up with GitHub"**，用刚才注册的 GitHub 账号直接登录，这样最方便
3. 按照提示授权即可

> 如果已经有账号了，直接用就行，跳过这步。

---

## 第一步：把代码上传到 GitHub

这一步的目的是：把你电脑上的工作台代码，上传到 GitHub 这个"代码网盘"里。

### 1.1 在 GitHub 上创建一个"仓库"

"仓库"就是 GitHub 上存放代码的一个文件夹。我们来创建它。

1. 在浏览器打开 https://github.com/new
2. 你会看到一个创建仓库的页面，需要填写以下内容：

   - **Repository name**（仓库名称）：填写 `shen-yaxu-workbench`
   - 勾选 **"Add a README file"**（添加一个说明文件）
   - 其他选项不用管，保持默认即可

3. 点击绿色的 **"Create repository"** 按钮

创建成功后，你会看到一个新页面，暂时不用管它，继续下一步。

---

### 1.2 下载并安装 GitHub Desktop

GitHub Desktop 是一个软件，它可以帮我们很方便地把电脑上的文件上传到 GitHub，不需要敲命令。

1. 在浏览器打开 https://desktop.github.com
2. 点击 **"Download for Windows"**（因为你是 Windows 电脑）下载安装包
3. 下载完成后，双击安装包，一路点"下一步"完成安装
4. 安装完成后，打开 GitHub Desktop
5. 在登录界面，选择 **"Sign in to GitHub.com"**，用你的 GitHub 账号登录
6. 登录成功后，按照提示填写你的名字和邮箱（随便填就行）

---

### 1.3 把仓库"克隆"到电脑上

"克隆"就是把 GitHub 上的仓库下载一份到你的电脑上。

1. 在 GitHub Desktop 顶部菜单栏，点击 **File** -> **Clone Repository**
2. 在弹出的窗口中，选择 **GitHub.com** 标签
3. 在列表里找到 `shen-yaxu-workbench`（就是你刚才创建的仓库）
4. **Local Path**（本地路径）：点击 "Choose..." 选择一个你找得到的位置，比如桌面或者文档文件夹
5. 点击 **"Clone"** 按钮

现在你的电脑上就有了一个叫 `shen-yaxu-workbench` 的文件夹。

---

### 1.4 把工作台文件复制进去

1. 找到你电脑上的 `online-workbench` 文件夹（就是包含工作台代码的那个文件夹）
2. 打开它，**全选**里面的所有文件和文件夹（按 `Ctrl+A`），然后复制（按 `Ctrl+C`）

   > 你应该能看到这些文件：`server.js`、`package.json`、`public` 文件夹、`render.yaml` 等。

3. 打开刚才克隆下来的 `shen-yaxu-workbench` 文件夹
4. **全选**里面的所有内容（按 `Ctrl+A`），然后删除（按 `Delete`）
   > 别担心，这些文件在 GitHub 上还有一份，删了也没关系
5. 把刚才复制的工作台文件**粘贴**进来（按 `Ctrl+V`）

现在，`shen-yaxu-workbench` 文件夹里应该全是工作台的代码文件了。

---

### 1.5 提交并推送到 GitHub

"提交"就是告诉 GitHub："我的文件有更新了，请保存一下"。

1. 打开 GitHub Desktop
2. 你应该能看到左侧显示了所有更改的文件（变绿色的那些）
3. 在左下角有一个输入框（**Summary**），填写：`添加工作台代码`
4. 点击左下角的蓝色 **"Commit to main"** 按钮
5. 紧接着，点击顶部的 **"Push origin"** 按钮

> "Push" 就是把电脑上的文件上传到 GitHub。等进度条跑完就成功了！

6. 你可以刷新一下浏览器中的 GitHub 仓库页面，确认文件已经上传上去了

**到这里，第一步就完成了！** 代码已经安全地放在 GitHub 上了。

---

## 第二步：在 Render 上部署

这一步的目的是：让 Render 把你 GitHub 上的代码"运行"起来，变成一个可以访问的网站。

### 2.1 登录 Render

1. 在浏览器打开 https://dashboard.render.com
2. 如果还没有登录，用 GitHub 账号登录（点击 "Sign in with GitHub"）

---

### 2.2 创建一个新的 Web Service

1. 登录后，点击右上角的 **"New +"** 按钮
2. 在下拉菜单中选择 **"Web Service"**

---

### 2.3 连接你的 GitHub 仓库

1. 你会看到一个连接 GitHub 的页面
2. 找到并点击 **"shen-yaxu-workbench"** 仓库
3. 点击蓝色的 **"Connect"** 按钮

---

### 2.4 设置部署参数

连接成功后，需要填写一些配置。**请严格按照下面的内容填写：**

| 设置项 | 填写内容 |
|--------|----------|
| **Name** | `shen-yaxu-workbench`（通常已自动填好） |
| **Runtime** | 选择 **Node.js** |
| **Build Command** | 填写 `npm install` |
| **Start Command** | 填写 `npm start` |
| **Instance Type** | 选择 **Free**（免费的） |

**其他选项保持默认，不用修改。**

---

### 2.5 添加环境变量

环境变量就是给程序传递一些"秘密信息"，比如密码密钥。

1. 在页面向下滚动，找到 **"Environment Variables"**（环境变量）部分
2. 点击 **"Add Environment Variable"** 按钮
3. 添加一个变量：
   - **Key** 填写：`JWT_SECRET`
   - **Value** 填写：你需要先生成一个随机字符串

**如何生成随机字符串：**

1. 打开 https://www.random.org/strings
2. 在页面上设置：
   - **Number of random strings to generate**：填 `1`
   - **Number of characters per string**：填 `32`
   - 其他选项保持默认
3. 点击 **"Get Strings"** 按钮
4. 复制生成的那个随机字符串（类似 `aB3x9Kp2mN7qR...` 这样的一串字符）
5. 把它粘贴到 Render 的 Value 输入框里

> 这个字符串就像一把锁的钥匙，程序用它来保护你的登录安全。不要告诉别人。

**关于 PORT 变量：** 如果你在 `render.yaml` 文件中已经设置了 PORT，Render 会自动识别，不需要手动添加。

---

### 2.6 开始部署

1. 确认以上所有设置无误后，滚动到页面底部
2. 点击蓝色的 **"Create Web Service"** 按钮

接下来，Render 会自动开始部署（就是帮你把网站搭建起来）。你会看到一个页面显示部署进度和各种日志。

**耐心等待 2-5 分钟**，这个过程不需要你做任何操作。

---

### 2.7 部署完成

部署成功后，页面顶部会显示一个绿色的 **"Live"** 标签。

在这个页面上方，你会看到一个网址，格式类似：

```
https://shen-yaxu-workbench.onrender.com
```

**这就是你的工作台网址！** 把它记下来，或者复制保存到手机上。

> 如果你看到一个 "Live" 标签旁边显示红色或黄色，说明部署可能出了问题。请联系技术人员帮忙看看。

---

## 第三步：使用工作台

现在你的工作台已经可以访问了！

### 3.1 打开网站

- 在电脑浏览器中，输入你刚才得到的网址（比如 `https://shen-yaxu-workbench.onrender.com`）
- 或者在手机浏览器中输入同样的网址

> 第一次打开可能需要等 30 秒左右，因为网站从"休眠"状态"醒来"需要一点时间。后面再打开就快了。

---

### 3.2 登录

1. 在登录页面，输入以下默认账号：

   - **用户名**：`admin`
   - **密码**：`123456`

2. 点击"登录"按钮

---

### 3.3 修改密码（重要！）

**首次登录后，请立即修改密码！** 默认密码太简单，不安全。

1. 登录成功后，在页面左侧边栏的最底部，找到 **"修改密码"** 按钮
2. 点击它
3. 输入你的旧密码（`123456`）
4. 输入你的新密码（建议用一个你记得住的、包含字母和数字的密码）
5. 确认新密码
6. 点击"保存"

---

## 第四步：导入已有数据

如果你之前在本地（自己电脑上）使用过工作台，并且导出过备份文件，可以把它导入到线上版本中。

### 4.1 导入备份

1. 登录工作台后，在侧边栏找到 **"导入备份"** 按钮
2. 点击它
3. 在弹出的文件选择窗口中，找到你之前导出的 JSON 备份文件（文件名通常类似 `backup_2024xxxx.json`）
4. 选中它，点击"打开"
5. 选择 **"合并模式"**（这样不会覆盖已有数据，只会把新数据加进来）
6. 等待导入完成，页面会提示成功

> 如果你没有之前的备份数据，跳过这步就行。

## 注意事项

请仔细阅读以下内容，避免以后遇到困惑。

### 1. 网站会"打瞌睡"

Render 免费版有一个特点：**如果 15 分钟内没有人访问你的网站，它会自动进入"休眠"状态**（相当于关机省电）。

- 当你再次打开网址时，需要等 **30 秒左右**，网站才会"醒来"
- 这期间页面可能显示错误或加载缓慢，**这是正常的，请耐心等待**
- 唤醒后，网站就能正常使用了

> 如果你不希望网站休眠，可以升级到 Render 的付费版，但免费版对我们的日常使用来说完全够用了。

### 2. 添加书签

建议你把工作台网址添加到手机浏览器的书签（收藏夹）里，这样随时都能快速打开：

- **手机（Chrome 浏览器）**：打开网址后，点击浏览器右上角的三个点 -> 点击书签图标（小旗帜）-> 确认添加
- **手机（Safari 浏览器）**：打开网址后，点击底部分享按钮（方框+箭头）-> 选择"添加书签"
- **电脑**：按 `Ctrl+D`（Windows）添加书签

### 3. 定期备份

虽然是线上版本，但为了数据安全，**建议每周导出一次备份**：

1. 登录工作台
2. 在侧边栏找到 **"导出备份"** 按钮
3. 点击它，会自动下载一个 JSON 文件到你的电脑/手机
4. 把这个备份文件保存好（可以放到网盘、U 盘等地方）

### 4. 忘记密码怎么办

如果忘记密码，暂时没有办法自助重置，需要联系技术人员帮忙处理。所以请务必记住你修改后的密码，最好把它记在一个安全的地方。

## 常见问题

**Q：打开网址后一直加载不出来怎么办？**

A：可能是网站正在"唤醒"中，请等待 30 秒后刷新页面试试。如果等了很久还是不行，可能是 Render 服务暂时不稳定，过一会再试。

**Q：我修改了代码，怎么更新到网上？**

A：在 GitHub Desktop 中修改文件后，再次执行 Commit 和 Push，Render 会自动检测到更新并重新部署（大约 2-5 分钟）。

**Q：免费版有什么限制？**

A：Render 免费版每月有 750 小时的运行时间，对于个人使用完全足够。主要限制就是 15 分钟无访问会休眠，以及偶尔可能因为资源紧张导致部署失败（重新部署即可）。

**Q：我可以用自己的域名吗？**

A：Render 免费版不支持自定义域名。如果需要，可以升级到付费版。
